Aquavital – Dehydration Risk Monitoring Demo System
===================================================

Purpose:
--------
A simple, browser‑based demo system that simulates dehydration‑risk monitoring for elderly individuals. 
The system consists of two separate HTML pages:
1. patient.html – generates simulated physiological readings and writes them to Firebase.
2. caregiver.html – displays live readings, alerts, and risk status from Firebase.

No build step. No npm. No bundlers. 
Everything must run from plain HTML + Firebase CDN scripts.

Firebase:
---------
Use Firebase Realtime Database via CDN:
<script src="https://www.gstatic.com/firebasejs/8.10.1/firebase-app.js"></script>
<script src="https://www.gstatic.com/firebasejs/8.10.1/firebase-database.js"></script>

Firebase config is embedded directly in both HTML files.
Database structure:

/readings/latest
    temperature: number
    pulse: number
    bis: number
    risk: number (0–1)
    risk_category: "green" | "red"
    timestamp: number (ms)

/alerts/{alertId}
    message: string
    timestamp: number

Patient Simulator (patient.html):
---------------------------------
• Generates random readings every 3 seconds:
    - Temperature (°C)
    - Pulse (bpm)
    - BIS (hydration index)
• Computes a risk score (0–1)
• Determines risk_category:
    - "green" for normal
    - "red" for dehydration risk
• Writes readings to Firebase under /readings/latest
• When risk_category becomes "red", writes an alert to /alerts

Caregiver Dashboard (caregiver.html):
-------------------------------------
• Reads /readings/latest in real time
• Displays:
    - Temperature
    - Pulse
    - BIS
    - Risk score
    - Last update time
• Shows a colored status indicator:
    - Green circle + text "Normal"
    - Red circle + text "Risk"
• Displays alert popups at the top‑center of the screen when new alerts appear
• Alert popup auto‑hides after 5 seconds

UI Requirements:
----------------
Landing page (index.html):
    • Provides two buttons:
        - “Patient Simulator”
        - “Caregiver Dashboard”

Caregiver UI:
    • Status circle must switch between green and red based on risk_category
    • Status text must switch between “Normal” and “Risk”
    • Alert popup must appear centered at the top of the screen

Patient UI:
    • Simple interface with a “Start Simulation” button
    • Shows current simulated values

Technical Constraints:
----------------------
• No frameworks
• No npm
• No imports
• No build tools
• Must run as static HTML files
• Must work on GitHub Pages
• Must use Firebase via CDN scripts only

Goal:
-----
Provide a clean, functional demo that simulates dehydration monitoring and demonstrates real‑time communication between two browser clients using Firebase.
